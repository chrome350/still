# Changelog — Stiletto Compute SPA (Figma plugin)

Все важные изменения в плагине ведём здесь, чтобы при передаче следующему
разработчику (или агенту-помощнику) контекст не терялся. Формат — обратный
хронологический (свежее сверху).

> Файлы плагина:
> - `manifest.json` — манифест Figma-плагина
> - `code.js` — sandbox-часть (создаёт прямоугольник, заливает картинкой, хранит auth в `clientStorage`)
> - `ui.html` — весь UI + JS (state, auth, генерация, экраны)
> - `assets/` — исходники картинок (для удобства); в рантайме всё, что показывается, **уже зашито** в `ui.html` как base64 data URI, т.к. UI Figma-плагина бежит в sandboxed iframe и не грузит относительные пути с диска.

### Тамбы стилей в popup — починили отображение
**Проблема:** в Figma-плагине UI крутится в sandbox iframe, поэтому `<img src="assets/embedded-NN.png">` (относительные пути) не загружаются. У 4 тамбов (`STYLE_PRESENTATION[].thumb`) висели именно такие пути → плагин показывал пустые плейсхолдеры.

**Фикс:** все 4 тамба теперь зашиты в `ui.html` как `data:image/png;base64,...`. Скрипт-инлайнер отрабатывает по ключам `objects/characters/animals/adt`. Если нужно обновить картинки:
1. Положить новый PNG в `assets/` (мы используем рабочие копии `embedded-11..14.png`, исходники под русскими именами тоже есть: `Объекты.png`, `Персонажи.png`, `Животные.png`, `Хромированая (ADT).png`).
2. Перепаковать base64 в `ui.html` (см. одноразовый Python-скрипт в истории чата — заменяет `data:image/png;base64,…` для нужного ключа).

### Поведение инпута во время генерации
**Главный триггер — `isGenerating`**, не наличие картинки в инпуте:
- `updateSendButton()` теперь корректно ставит класс `is-generating` на `.btn-send` и `.input-bar`. До этого функция дёргала несуществующие селекторы `.main-img / .pause-img` и фактически ничего не переключала — стоп-иконка вообще не показывалась.
- CSS-правила:
  - `.input-bar.is-generating .style-dropdown-wrapper { display: none; }` — выбор стиля прячется
  - `.input-bar.is-generating .btn-send .send-icon → none, .pause-icon → block` — кнопка превращается в стоп

### Поведение инпута с прикреплённой картинкой (вторичный триггер)

### Стоп-иконка вместо паузы
SVG `.pause-icon` (две вертикальные полосы) → одна **квадратная заливка** 8×8 (стоп). Сам класс называется `pause-icon` для обратной совместимости с CSS-правилами, но визуально это стоп.

### Done screen — картинка вместо стикера
Изначально на done-экране было нарисовано verified-check (SVG-стикер). Заменили на `<img id="done-image">` 80×80 — туда подставляется **та же картинка**, которую показала плашка загрузки и которая ушла на канвас Figma. Это работает и в DEV, и в реальном пути.

Для корректного MIME (PNG в реале / JPEG в DEV) добавлена переменная `generatedImageDataUri` рядом с `generatedImageBase64`. Полный data-URI хранится отдельно от чистого base64; чистый base64 идёт в `code.js` через `fill-placeholder`, data-URI — в `<img>`.

### DEV-режим (тестовая версия без бэка)
Добавлен флаг **`const DEV_MODE = true;`** в начале `<script>` блока `ui.html` (~стр. 1116) и словарь `FAKE_GENERATION_IMAGES` (4 JPEG-картинки из `assets/Fake generation/`, сжатые до ~1024px и зашитые base64).

При `DEV_MODE = true`:
1. На бутстрапе скрипта **обходим SSO**: вместо `requestAuthStateFromMain()` + `setAppState('auth-required')` сразу заполняется фейковый `commandsConfig` (4 стиля) и идёт `setAppState('start')`. Это убирает «мигание» auth-required → start при старте плагина.
2. `startGeneration()` редиректит на `startFakeGeneration()`:
   - 2 сек шиммер
   - подставляет картинку под текущий стиль и в loading-плашку (через `has-image`), и на канвас Figma (через `fill-placeholder`)
   - 0.9 сек паузы
   - `setAppState('done')`

**Чтобы выкатить на прод-API**: поменять `DEV_MODE = false`. Все остальные изменения (квадрат, тексты, done-экран, поведение инпута) работают в обоих режимах одинаково. В реальной генерации тоже добавлен показ результата в loading-плашке (`has-image`) перед переходом на done — для визуальной парности с DEV.

## Как тестировать локально

1. Открыть Figma (десктоп).
2. Plugins → Development → Import plugin from manifest… → выбрать `manifest.json` из этой папки.
3. Запустить «Stiletto Compute SPA» из Plugins → Development.
4. В DEV-режиме SSO не требуется: ввести любой текст, нажать отправить → через 2 сек появится фейковая картинка и в плашке, и на канвасе.

## Известные нюансы для следующего разработчика

- **Sandbox iframe**: ничего не грузить через относительные пути из `ui.html` — только data URI / `figma.createImage` внутри `code.js`.
- **Размер `ui.html`**: после инлайна фейковых JPEG’ов файл ~640 KB. Не критично, но при добавлении новых ассетов следить.
- **Старт без auth**: убедиться, что `setAppState('auth-required')` остаётся для прод-режима и что `requestAuthStateFromMain()` срабатывает только при `DEV_MODE = false`.
- **Стоп-иконка при прикреплённом изображении**: сейчас это **только визуал** (CSS меняет иконку). Поведение клика не менялось — кнопка по-прежнему вызывает `handleSend()` (старт генерации). Если нужно реальное «стоп» при наличии картинки — править `handleSend()`.
